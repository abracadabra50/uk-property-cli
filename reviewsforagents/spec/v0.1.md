# reviewsforagents.md Specification v0.1

## Overview

A standard for exposing review data in an agent-optimized markdown format.
Any business can serve a reviewsforagents.md endpoint. This spec defines the format.

## Transport

- HTTP GET request
- Response Content-Type: `text/markdown` (or `application/json` for `?format=json`)
- UTF-8 encoded

## URL Structure

| Pattern | Description |
|---|---|
| `/{domain}` | Full store/business summary (standard format) |
| `/{domain}/{product-or-service}` | Specific item detail (detailed format) |
| `/{domain}?format=min` | Minified format for bulk comparison |
| `/{domain}?format=json` | Structured JSON alternative |

## Output Formats

### Standard (~200-300 tokens)

```markdown
# {domain} | {rating}★ | {count} reviews | {verified_pct}% verified

## Top Products
{product} | {rating}★ ({count}) | {theme summary}

## Sentiment
+{positive} +{positive}
-{negative} -{negative}

## Trust
Response rate: {pct}% | Avg response: {time} | Return mentions: {pct}%
```

### Minified (~50-100 tokens)

```
{domain};{rating};{count};{verified_pct}%v|{handle}:{rating}:{count}:+{pos}-{neg}
```

### Detailed (~500-800 tokens)

```markdown
# {Product} — {domain} | {rating}★ | {count} reviews

## Rating Distribution
5★ ████████████████ {pct}%
4★ ██████████ {pct}%
...

## Key Themes
+{theme} (mentioned {count}x) | -{theme} ({count}x)

## Recent Highlights (last 30 days)
- {rating}★ "{quote}" — verified

## Fit Guide (from reviews)
True to size: {pct}% | Runs small: {pct}% | Runs large: {pct}%
```

### JSON

Structured JSON containing the same data points as the standard format.

## Required Fields (all verticals)

| Field | Type | Description |
|---|---|---|
| Entity name / domain | string | The business domain |
| Overall rating | number | Normalized to 5-point scale |
| Total review count | integer | Total reviews analyzed |
| Source attribution | string | Which platforms reviews came from |

## Optional Fields (all verticals)

| Field | Type | Description |
|---|---|---|
| Verified percentage | number | % of reviews from verified purchasers |
| Rating distribution | object | 1-5 star breakdown |
| Sentiment tags | string[] | `+positive` / `-negative` theme tags |
| Trust signals | object | Response rate, response time |
| Recent review highlights | array | Representative recent reviews |

## Token Budget

| Format | Target Tokens |
|---|---|
| Standard | 200-300 |
| Minified | 50-100 |
| Detailed | 500-800 |
| JSON | N/A (structured data) |

## Response Headers

| Header | Values | Description |
|---|---|---|
| `X-Cache` | `HIT` / `MISS` | Whether response was served from cache |
| `X-Source` | e.g., `judgeme` | Comma-separated list of review platforms used |
| `X-Reviews-Count` | integer | Total reviews analyzed |
| `X-Generated` | ISO 8601 | When the summary was generated |
| `X-Spec-Version` | `0.1` | Specification version |

## Cache Behavior

- Default TTL: 6 hours
- Unsupported domains: 1 hour TTL
- Cache key includes: domain + product handle + format

## Rate Limits

- Free tier: 100 requests/hour per IP
- Responses include standard rate limit headers
